Classes

Use JuDoc's classes or add your own.

By now you know that you can add @@divname ... @@ anywhere you want in your content in order to define an environment and style it. Below we detail the default classes as well as some tricks that can be useful to style your content.

JuDoc's classes

NameFunction
jd-contentwraps around the content of a page, so when a mypage.md is converted to some html, it gets inserted in the scaffolding as <div class="jd-content"> some html </div>
jd-tocwraps around the inserted table of contents
fnrefwraps around a footnote reference
fndefwraps around footnote definitions
fndef-contentwraps around the content of footnote definitions
fndef-backrefwraps around the back-reference of a footnote definition

Simple tricks

As soon as you're using a style more than once, it makes sense to create it as a command. This makes it much easier to maintain and re-use. For instance, consider the following simple examples:

\newcommand&#123;\blurb&#125;&#91;1&#93;&#123;
    ~~~
    &lt;span style&#61;&quot;font-size:24px;font-weight:300;&quot;&gt;&#33;#1&lt;/span&gt;
    ~~~
&#125;
\newcommand&#123;\note&#125;&#91;1&#93;&#123;@@note @@title ⚠ Note@@ @@content #1 @@ @@&#125;

Further, you can pass the style as an argument:

\newcommand&#123;\spstyle&#125;&#91;2&#93;&#123;~~~&lt;span style&#61;&quot;#1&quot;&gt;#2&lt;/span&gt;~~~&#125;

and for instance:

\spstyle&#123;font-variant:small-caps;font-size:15px;color:cornflowerblue&#125;&#123;formatted text&#125;

gives you: formatted text.
